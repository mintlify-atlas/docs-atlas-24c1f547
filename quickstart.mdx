This guide walks you through authenticating with Unikraft Cloud and deploying your first unikernel instance.

## Prerequisites

Before you begin, ensure you have:

- [Installed the Unikraft CLI](/installation)
- A [Unikraft Cloud account](https://console.unikraft.cloud/auth/signin)

## Step 1: Authenticate with Unikraft Cloud

<Steps>
  <Step title="Login to Unikraft Cloud">
    Run the login command to authenticate:

    ```bash
    unikraft login
    ```

    This command will:
    1. Open your default browser automatically
    2. Prompt you to authenticate via the Unikraft Cloud console
    3. Wait for confirmation (default timeout: 5 minutes)

    <Tip>
    If your browser doesn't open automatically, copy and paste the URL shown in the terminal.
    </Tip>
  </Step>

  <Step title="Verify authentication">
    Once authenticated, you'll see a confirmation message:

    ```
    authenticated organization=<your-org-name>
    login successful profile=<your-org-name>
    ```

    Check your authentication status anytime:

    ```bash
    unikraft login --check
    ```
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Alternative: Login with token file">
    If you prefer to use a token file instead of browser authentication:

    ```bash
    unikraft login --token /path/to/token
    ```

    Or read from stdin:

    ```bash
    unikraft login --token -
    ```

    When using a token file, you must also specify your organization:

    ```bash
    unikraft login --token /path/to/token --organization my-org
    ```
  </Accordion>

  <Accordion title="Alternative: Manual profile configuration">
    For advanced use cases, you can manually create a profile:

    ```yaml
    # Linux: ~/.config/unikraft/config.yaml
    # MacOS: ~/Library/Application Support/unikraft/config.yaml
    profile: default
    profiles:
      default:
        type: cloud
        organization: <org-name>
        token: <api-token>
        metros:
          - name: fra
            endpoint: https://api.fra.unikraft.cloud
            country: de
            insecure: false
    ```

    <Warning>
    Avoid setting `insecure: true` in production environments as it skips TLS verification.
    </Warning>
  </Accordion>
</AccordionGroup>

## Step 2: List Available Metros

View all available deployment locations:

```bash
unikraft metros list
```

Example output:

```
NAME  COUNTRY  ENDPOINT
fra   de       https://api.fra.unikraft.cloud
sfo   us       https://api.sfo.unikraft.cloud
sin   sg       https://api.sin.unikraft.cloud
was   us       https://api.was.unikraft.cloud
dal   us       https://api.dal.unikraft.cloud
```

<Tip>
Choose a metro closest to your users for the best performance. Use the metro `name` in deployment commands.
</Tip>

## Step 3: Deploy Your First Instance

Deploy an NGINX instance with HTTPS enabled:

```bash
unikraft run --metro=fra -p 443:8080/http+tls nginx:latest
```

This command:
- Deploys an NGINX unikernel in Frankfurt (`fra`)
- Exposes port 8080 as HTTPS on port 443
- Automatically provisions a TLS certificate
- Starts the instance immediately

<Note>
The instance will boot in **milliseconds** and be ready to serve traffic.
</Note>

### Understanding the Run Command

The `unikraft run` command accepts several options:

| Option | Description | Example |
|--------|-------------|---------|
| `--metro` | Deployment location (required) | `--metro=fra` |
| `-p, --publish` | Expose service ports | `-p 443:8080/http+tls` |
| `-e, --env` | Set environment variables | `-e KEY=VALUE` |
| `-v, --volume` | Attach persistent volumes | `-v my-vol:/data` |
| `-m, --memory` | Memory allocation | `-m 512MiB` |
| `--vcpus` | Number of vCPUs | `--vcpus 2` |
| `-n, --name` | Instance name | `-n my-app` |

<Tip>
For a complete list of options, run `unikraft run --help`.
</Tip>

## Step 4: Verify Your Deployment

<Steps>
  <Step title="List your instances">
    View all running instances:

    ```bash
    unikraft instances list
    ```

    Example output:

    ```
    METRO  NAME                STATE    IMAGE          MEMORY   VCPUS  CREATED
    fra    nginx-abc123        running  nginx:latest   128MiB   1      2s ago
    ```
  </Step>

  <Step title="Get detailed instance information">
    Inspect a specific instance:

    ```bash
    unikraft instances get <instance-name>
    ```

    This shows detailed information including:
    - Service URLs and domains
    - Resource allocation
    - Runtime configuration
    - Network details
  </Step>

  <Step title="View instance logs">
    Monitor your instance logs in real-time:

    ```bash
    unikraft instances logs <instance-name> --follow
    ```

    Or fetch the last 100 lines:

    ```bash
    unikraft instances logs <instance-name> --tail 100
    ```
  </Step>
</Steps>

## Step 5: Deploy with Advanced Features

Now that you've deployed a basic instance, try these common deployment patterns:

<CodeGroup>
```bash Environment Variables
unikraft run --metro=sfo \
  -e DATABASE_URL=postgres://... \
  -e ENVIRONMENT=production \
  -e DEBUG=false \
  my-app:latest
```

```bash Persistent Storage
unikraft run --metro=was \
  -v my-data-volume:/data \
  -v logs-volume:/var/log \
  my-app:latest
```

```bash Custom Resources
unikraft run --metro=dal \
  -m 512MiB \
  --vcpus 2 \
  my-app:latest
```

```bash Scale-to-Zero
unikraft run --metro=sin \
  --scale-to-zero policy=on,cooldown-time=300 \
  my-app:latest
```

```bash HTTP with HTTPS Redirect
unikraft run --metro=fra \
  -p 443:8080/http+tls \
  -p 80:443/http+redirect \
  nginx:latest
```
</CodeGroup>

### Working with Volumes

Create a persistent volume before attaching it:

<Steps>
  <Step title="Create a volume">
    ```bash
    unikraft volumes create \
      --set name=my-data \
      --set metro=fra \
      --set size=1024
    ```
  </Step>

  <Step title="Deploy with the volume">
    ```bash
    unikraft run --metro=fra \
      -v my-data:/data \
      my-app:latest
    ```
  </Step>

  <Step title="Attach as read-only">
    ```bash
    unikraft run --metro=fra \
      -v my-data:/data:ro \
      my-app:latest
    ```
  </Step>
</Steps>

## Managing Your Instances

Common instance management operations:

<CodeGroup>
```bash Start Instance
unikraft instances start <instance-name>
```

```bash Stop Instance
unikraft instances stop <instance-name>
```

```bash Restart Instance
unikraft instances restart <instance-name>
```

```bash Delete Instance
unikraft instances delete <instance-name>
```

```bash Update Instance Image
unikraft instances edit <instance-name> --set image=nginx:1.27
```

```bash Scale Memory
unikraft instances edit <instance-name> --set resources.memory=256
```
</CodeGroup>

<Warning>
Deleting an instance is irreversible. Ensure you have backups of any important data stored in attached volumes.
</Warning>

</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Instance fails to start">
    If your instance fails to start:
    
    1. Check the instance logs: `unikraft instances logs <instance-name>`
    2. Verify the image exists and is compatible
    3. Ensure sufficient resources are available in the metro
    4. Check service port configurations are valid
  </Accordion>

  <Accordion title="Cannot connect to instance">
    If you cannot reach your deployed instance:
    
    1. Verify the instance is in `running` state
    2. Check service group and domain configuration
    3. Ensure firewall rules allow traffic to the exposed ports
    4. Verify TLS certificate provisioning completed successfully
  </Accordion>

  <Accordion title="Authentication expired">
    If you receive authentication errors:
    
    1. Check your login status: `unikraft login --check`
    2. Re-authenticate: `unikraft login`
    3. Verify your organization membership hasn't changed
  </Accordion>

  <Accordion title="Volume attachment errors">
    If volume attachment fails:
    
    1. Verify the volume exists in the same metro
    2. Ensure the volume isn't already attached to another instance
    3. Check the mount path is valid and doesn't conflict with system paths
    4. Confirm volume size is sufficient
  </Accordion>
</AccordionGroup>

## Environment Variables

Customize CLI behavior with environment variables:

| Variable | Description | Example |
|----------|-------------|---------|
| `UNIKRAFT_CONFIG` | Configuration file path | `~/.config/unikraft/config.yaml` |
| `UNIKRAFT_PROFILE` | Override active profile | `production` |
| `UNIKRAFT_LOG_LEVEL` | Log verbosity | `trace`, `debug`, `info`, `warn`, `error` |
| `UNIKRAFT_LOG_TYPE` | Output format | `text`, `json` |
| `UNIKRAFT_TELEMETRY` | Enable/disable telemetry | `true`, `false` |

Example usage:

```bash
UNIKRAFT_LOG_LEVEL=debug unikraft instances list
```

<Note>
The CLI collects anonymous usage analytics to improve the product. No personally identifiable information is collected. Opt out by setting `UNIKRAFT_TELEMETRY=false` or `DO_NOT_TRACK=1`.
</Note>
